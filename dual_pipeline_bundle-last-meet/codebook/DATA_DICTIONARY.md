# Data Dictionary

## Field definitions

- `Passage_Name`: Passage display name from report metadata.
- `passage_id`: Passage identifier.
- `Transition sentence`: Last sentence of the chunk transcript — cue for when to show this chunk’s image during playback.
- `chunk_start_seconds`: Chunk start time in seconds from the beginning of the audio.
- `chunk_end_seconds`: Chunk end time in seconds from the beginning of the audio.
- `chunk_duration_seconds`: Chunk duration in seconds (chunk_end_seconds - chunk_start_seconds).
- `chunk_start_mmss`: Chunk start time formatted as MM:SS.s.
- `chunk_end_mmss`: Chunk end time formatted as MM:SS.s.
- `chunk_duration_mmss`: Chunk duration formatted as MM:SS.s.
- `chunk_id`: Chunk index within passage.
- `prompt_id`: Prompt identifier (blank when unavailable).
- `prompt_text`: Prompt text used for image generation.
- `image_index`: Evaluated image index for this row.
- `URL`: Relative image path in bundle (under images/).
- `Final Image Index`: Final selected image index reference.
- `ai_consistency`: AI-evaluator consistency score (0-10), when applicable.
- `ai_clip_similarity`: AI-reported CLIP similarity value.
- `ai_ocr_keyword`: AI-evaluator OCR keyword result text.
- `ai_speaker_verification`: AI-evaluator speaker verification status.
- `ai_audio_image_match`: AI score for transcript-to-image semantic match (0-10).
- `ai_coherence`: AI coherence score (0-10).
- `ai_signaling`: AI signaling/focus score (0-10).
- `ai_text_quality`: AI text quality score (0-10).
- `ai_emotional_design`: AI emotional design score (0-10).
- `ai_total_weighted_score`: AI total weighted score (0-10).
- `system_clip_similarity`: CLIP similarity raw value from system evaluator.
- `system_clip_similarity_score`: Scaled CLIP similarity score from system evaluator.
- `system_brightness`: Image brightness metric.
- `system_contrast`: Image contrast metric.
- `system_sharpness`: Image sharpness metric.
- `system_colorfulness`: Image colorfulness metric.
- `system_attractiveness`: Image attractiveness/aesthetic metric.
- `system_mean_saturation`: Chromatic metric: mean HSV saturation S̄, scaled 0–10 (Table 1).
- `system_saturation_contrast`: Chromatic metric: σ(S) over pixels, scaled 0–10.
- `system_hue_singularity`: Chromatic metric: 1−H(hue)/log₂(K) on S>0.15 pixels, K=36, scaled 0–10.
- `system_chromatic_salience`: Chromatic metric: fraction of pixels with S>0.40 (τ), scaled 0–10.
- `system_predicted_character_count`: Predicted number of people in image.
- `system_expected_character_count`: Expected number of people from prompt/audio context.
- `system_count_alignment_score`: Count alignment score between expected and predicted characters.
- `system_expected_speaker_phrase`: System-generated expected speaker phrase summary.
- `system_gender_similarity`: Gender similarity metric.
- `system_gender_alignment_score`: Gender alignment score metric.
- `system_keyword_mode`: Keyword mode used in evaluation (visible-natural or hidden).
- `system_detected_keyword`: Detected keyword text from OCR/system checks.
- `system_exact_letter_match_case_insensitive`: Whether detected keyword exactly matches expected (case-insensitive).
- `system_keyword_missing_visual`: Flag indicating expected visible keyword was missing.
- `system_spatial_mismatch`: Flag indicating spatial mismatch penalty signal.
- `system_surface_content_mismatch`: Flag indicating surface-content mismatch penalty signal.
- `chunk_transcript`: Chunk-level transcript text.
- `full_transcript`: Passage-level full transcript text.
- `pipeline_type`: Source pipeline label (auto or simple).
- `job_id`: Job identifier from report metadata.
- `session_id`: Session identifier from report metadata.
- `name_en`: English passage name from report metadata.
- `missing_ai_eval_flag`: 1 if AI evaluation score missing; otherwise 0.
- `missing_system_metrics_flag`: 1 if key system metrics missing; otherwise 0.

## Notes
- Primary keys: `passage_id`, `chunk_id`, `pipeline_type`, `image_index`
- `URL` is relative to the bundle root (e.g. `images/auto_p1_c0_img1.png`)
- CSVs are under `csvs/panel_auto.csv` and `csvs/panel_simple.csv`
